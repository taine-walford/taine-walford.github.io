What are the best practices associated with Classes vs IDs?

First, what are Classes?

Classes are defined in HTML and used as a way to select an HTML element in CSS to designate styles to HTML elements. For instance, I could have a .red class that simply turns the text inside the element 'red'. I then apply this to an html element and voila - the text is red. This can be apllied to multiple elements.
In short terms, it allows for an amount of CSS to be easily latched to an element/s.

What are IDs?

IDs are defined in HTML, and can be used to designate an ID to an HTML element, however IDs must be unique to work correctly. IDs need to be unique within the page and are good for single elements that need to be selected in JS. This way an element can remain specific and easily 'identified' out of all the elements on the page.
In shorter terms, an ID is a license plate for each element, and obviously you'll run into trouble if you get caught with two people with the same license plate.

What do I do with all this powerful knowledge?

A very simple way to figure out which one you'll need is to ask yourself: "Is there any other element that will require the same style, now or any time in the future?"

If the answer is anything but a firm 'No', use a class.

The caveat to this, however, is that elements can have both an ID and a class (or even multiple). This way you can apply reused styles, but the element can have a specific ID tied to it, giving you the best of both worlds. As long you know exactly why you're using ID, you'll be fine. Otherwise, class functions exactly the same and without the mess.
